# Experience Report at AWS Community Day 2026

## Program Objectives

The event is an in-depth technical forum for engineers, system architects, and information technology students focusing on Operations and Artificial Intelligence (AI) on the AWS platform. The program focuses on process automation, building smart virtual assistants, and optimizing cloud infrastructure management.

Through the event, attendees have the opportunity to:

- Grasp the latest methods for integrating AI into incident response workflows and DevOps.
- Understand the architecture behind real-time Voice Agent systems.
- Update knowledge on setting up secure connections (VPC) for AI models.
- Network and learn practical system architecture deployment experiences from AWS experts.

---

# Summary of Presentation Sessions

## Session 1 - Deep Response Engine: From Detection to Autonomous Resolution

**Speaker:** Steve Tran - Cloud Operations Expert from AWS

### Content Highlights

- The difficulties and complex "barrier walls" in modern cloud system operations.
- The inevitable shift from alert-driven systems to action-driven systems.
- Architectural overview of the Deep Response Engine.
- Business impact: Cost reduction and moving towards the goal of zero-downtime operations.

### Key Takeaways

- Shifting the mindset from engineers manually fixing errors to building "self-healing" systems.
- Understanding the basic operational flow of a Response Engine in a cloud environment.
- Recognizing the value of automation in minimizing system downtime risks.

---

## Session 2 - Voice Agents: Building Human-Like AI Conversations at Scale

### Content Highlights

- The evolution of communication systems: from IVR (Interactive Voice Response) and traditional chatbots to AI voice agents.
- Solving core challenges: latency, accuracy, and naturalness in communication.
- Introduction to the Amazon Nova Sonic foundation model and speech-to-speech processing capabilities.
- System architecture: Combining telephony systems, streaming, Amazon Bedrock, and MCP tools.
- Enterprise use cases, best practices, and a visual live demo.

### Key Takeaways

- Clearly understanding the streaming data processing flow to minimize latency for language and voice models.
- How to combine AWS services (like Bedrock) to create the smoothest and most natural AI conversations.
- Grasping the architecture of integrating external tools into the reasoning process of a Voice Agent.

---

## Session 3 - AWS DevOps Agent: Your Always-Available Operations Teammate

### Content Highlights

- Overview of the AWS DevOps Agent tool.
- How to use AI to minimize Mean Time To Detect (MTTD) and Mean Time To Resolve (MTTR) incidents.
- Operations support capabilities in multi-cloud and hybrid cloud environments.
- Exploring Bedrock AgentCore and the multi-agent reasoning approach.
- Real-world use cases and an in-depth demo on the Amazon ECS environment.

### Key Takeaways

- Knowing how to apply an AI Agent as an "always-available teammate" to support 24/7 system monitoring.
- How to set up multi-agents to break down and solve complex DevOps problems.
- Grasping how to integrate monitoring tools directly into containers on ECS.

---

## Session 4 - AI-Powered Productivity: Workforce Planning For Enterprise

### Content Highlights

- Challenges in the digital transformation of the Human Resources (HR) sector in modern enterprises.
- Overview of the Amazon Quick platform and its specialized features for HR.
- Accelerating and automating HR operations.
- Workforce analytics and providing data-driven insights.
- Strategic workforce planning to support the decision-making process for management.

### Key Takeaways

- Understanding how AI is completely transforming the human resource management problem.
- How to turn raw data into visual reports and valuable insights using AI.
- The mindset of applying technology to solve internal operational problems (not just limited to customer-facing products).

---

## Session 5 - Building Secure Private MCP Connection with Amazon Quick

### Content Highlights

- Introduction to the Amazon Quick AI assistant platform.
- The role of MCP (Model Context Protocol) in extending the capabilities of AI models.
- Information security challenges when integrating via MCP.
- Guide on configuring Private Connectivity via AWS VPC for Amazon Quick.
- Demo and sharing of practical deployment experiences.

### Key Takeaways

- Clearly understanding the MCP to securely connect LLMs with external data sources.
- Mastering the principles of network security and access control for AI applications.
- Knowing how to set up a private network configuration (VPC) to ensure data is not leaked to the Internet.

# Conclusion After the Event

After attending the event, I gained a more comprehensive and practical perspective on how AI is reshaping cloud operations and software development processes. The presentation sessions provided a perfect combination of architectural theory and practice (demos), helping me better understand complex concepts.

Some outstanding knowledge I acquired includes:
- The importance of Autonomous Resolution rather than just passive monitoring.
- The foundational architecture of a real-time Voice Agent, addressing latency issues in voice communication via streaming and Bedrock.
- The power of Multi-Agent reasoning in optimizing DevOps workflows and reducing incident resolution time (MTTD/MTTR).
- The ability to apply the Amazon Quick platform for both business process automation (HR) and as an intelligent assistant.
- The strict security standards when establishing AI connections via MCP, especially knowledge regarding Private VPC configuration.
